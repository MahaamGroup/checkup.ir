# CHECKUP Project Architecture v1.0

## Purpose

This document defines the required project architecture.

It complements the Website Build Specification.

It does not change product behavior.

It defines only code organization.

------------------------------------------------------------------------

# Root Structure

    checkup.ir/

    ├── app/

    ├── components/

    ├── content/

    ├── lib/

    ├── hooks/

    ├── types/

    ├── styles/

    ├── public/

    ├── docs/

    ├── package.json

    ├── tsconfig.json

    ├── tailwind.config.ts

    └── next.config.ts

------------------------------------------------------------------------

# app/

Use Next.js App Router.

Recommended structure:

    app/

    layout.tsx

    page.tsx

    why-checkup/

    management-checkup/

    opportunity-paths/

    process/

    insights/

    insights/[slug]/

    case-studies/

    case-studies/[slug]/

    about/

    contact/

    privacy/

    terms/

    cookies/

    not-found.tsx

No page should contain large UI implementations.

------------------------------------------------------------------------

# hooks/

Reusable hooks only.

No page-specific hooks.

------------------------------------------------------------------------

# types/

Shared interfaces.

------------------------------------------------------------------------

# styles/

Global styles only.

Tailwind remains the primary styling system.

Avoid custom CSS unless absolutely necessary.

------------------------------------------------------------------------

# Naming Rules

Components

PascalCase

Files

PascalCase for components.

Directories

lowercase.

------------------------------------------------------------------------

# Imports

Prefer alias imports.

Avoid long relative paths.

------------------------------------------------------------------------

# Architecture Rules

Pages compose.

Components render.

Content supplies data.

Utilities support.

No architectural layer should violate these responsibilities.

------------------------------------------------------------------------

# Final Rule

The finished repository should resemble a professional enterprise
frontend codebase rather than a generated website.

This architecture document is mandatory.

It complements all previous project documents.
