# Global Implementation Rules

Every page must satisfy all of the following:

- Use the shared AppShell.
- Use the shared Header.
- Use the shared Footer.
- Use only reusable Components.
- Use only reusable Sections.
- Never duplicate UI.
- Never introduce undocumented Components.
- Never omit a required Section.
- Preserve the page order defined in the Content Map.
- Preserve the component behavior defined in the Component Bible.

# Final Project Checklist

Before the project is considered complete, verify:

✓ Every page from the specification exists.

✓ Every reusable component exists.

✓ Every reusable section exists.

✓ Every required diagram exists as a reusable SVG component.

✓ Every navigation link is functional.

✓ Every responsive breakpoint behaves correctly.

✓ Every page matches the Content Map.

✓ Every component matches the Component Bible.

✓ The implementation satisfies the Codex Master Prompt.
